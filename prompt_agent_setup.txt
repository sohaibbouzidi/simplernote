# External Memory Rule — Simplernote AI API Integration

You must use the external memory API at `http://localhost:8000/api/` as the persistent memory, task orchestration, and AI knowledge backend for all long-running workflows, planning, coding sessions, debugging, research, and multi-agent execution.

The API acts as the shared memory layer between sessions, agents, and workflows.

---

# CONFIGURATION REQUIREMENTS

Before any API usage, the following must be defined:

```bash id="cfg1"
SIMPLERNOTE_API_KEY="<API_KEY>"
PROJECT_ID="<PROJECT_ID>"
```

### Validation Rule

Before executing ANY API operation:

* If `SIMPLERNOTE_API_KEY` is missing → request it from the user
* If `PROJECT_ID` is missing → request it from the user
* If either is missing → stop execution immediately
* Never proceed with partial configuration

Example prompt:

```text id="cfg2"
Missing required configuration:
- SIMPLERNOTE_API_KEY
- PROJECT_ID

Please provide them before continuing.
```

---

# AUTHENTICATION

All requests must include:

```http id="auth1"
Authorization: Bearer <API_KEY>
Content-Type: application/json
```

Base URL:

```text id="base1"
http://localhost:8000/api/
```

---

# CORE RULES

## 1. Memory-First Execution

Before starting ANY task:

1. Search AI context knowledge base
2. Retrieve existing notes using PROJECT_ID
3. Retrieve existing tasks using PROJECT_ID
4. Reuse existing architecture, research, workflows

Never start from zero if prior context exists.

---

## 4. AI Context (Shared Memory Layer)

Endpoint:

```text id="ctx1"

```

* semantic memory retrieval
* session continuity
* architecture recall
* cross-agent coordination

Always query before generating plans or code.

# FAILURE HANDLING

If API fails:

1. continue locally
2. queue updates
3. retry later
4. sync when recovered

Never lose work.
